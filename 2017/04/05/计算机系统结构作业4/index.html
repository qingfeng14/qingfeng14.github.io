有一个4-way组相连Cache， Cache大小为16K字节(4K字), Cache块大小为16字节, 执行以下程序：

  1. 若 Cache 采用写分配（ Write allocate） 策略，计算执行整个程序时的 Cache 缺失率 写分配策略中写缺失时会先从Cache中把对应的块调到cache中，然后进行写操作，由于Cache块大小为16字节，每个int变量为4字节，进而在一开始的4096次写操作中，每4次写会发生一次缺失，即会发生1024次缺失，由于Cache大小为16k字节，完成第一轮循环时恰好将整个M数组载入Cache中，随后的9轮访存不会发生缺失，缺失率为

    \[miss\ rate = \frac{1024\ misses}{4096*10\ accessed} = 0.025\]

  2. 若 Cache 采用写不分配（ No-write allocate） 策略，计算执行整个程序时的 Cache 缺失率 采用写不分配策略时，发生写缺失时，会直接写回主存，进而在本程序中每次写操作都会发生缺失，进而缺失率为100%

  3. 针对写不分配的 Cache 缺失率高的问题，试对程序进行优化，以取得最小的 Cache 缺失 率。 给出优化后的程序代码，并计算优化后的 Cache 缺失率
